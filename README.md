# 基于联邦深度强化学习的边缘缓存原型系统仿真 (FedCaching-Sim)

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Python](https://img.shields.io/badge/python-3.8%2B-green.svg)](https://www.python.org/)
[![PyTorch](https://img.shields.io/badge/pytorch-1.10%2B-orange.svg)](https://pytorch.org/)

## 📖 概述

随着 5G/6G 网络及物联网技术的飞速发展，海量内容分发对核心网带宽造成了巨大压力。**边缘缓存 (Edge Caching)** 作为一种将内容预置于网络边缘（如基站、边缘服务器）的关键技术，能够显著降低内容获取时延并缓解骨干网拥塞。

本项目实现了一个**基于联邦深度强化学习 (Federated Deep Reinforcement Learning, FDRL)** 的边缘缓存原型仿真系统。该系统结合了深度强化学习（DQN）的决策能力与联邦学习（Federated Learning）的隐私保护及分布式协同优势，旨在动态优化边缘节点的缓存策略，以最大化长期缓存命中率（Cache Hit Ratio, CHR）。

## 🚀 核心特性

- **分布式协同决策**：利用联邦学习框架，各边缘节点（Client）在本地训练 DQN 模型，仅上传模型参数而非原始请求数据，有效保护用户隐私。
- **深度强化学习驱动**：采用卷积神经网络（CNN）提取内容请求的时空特征，通过 DQN 算法在海量内容空间中实现最优缓存替换决策。
- **动态环境感知**：仿真环境能够模拟真实的内容请求分布及其时变特性，支持多种参数配置（如缓存容量、请求频率、客户端数量等）。
- **高性能仿真架构**：基于 PyTorch 构建，支持 GPU 加速，提供完整的训练、聚合及评估流程。

## 🏗️ 系统架构

系统主要由以下核心组件构成：

1.  **Central Server (CS)**：负责全局模型的维护、分发以及本地模型参数的联邦聚合（如 FedAvg 算法）。
2.  **Base Stations (BS/Clients)**：作为边缘节点，负责收集本地用户请求、执行缓存决策、进行本地模型训练并上传更新。
3.  **Caching Environment**：模拟边缘缓存场景，包括内容流行度模型、缓存命中机制及奖励函数计算。

## 🛠️ 算法流程

1.  **初始化**：中央服务器初始化全局 DQN 模型参数。
2.  **本地训练**：各基站下载全局模型，根据本地观测到的内容请求序列，利用 DQN 算法进行多轮迭代训练。
3.  **参数上传**：基站将训练后的本地模型权重上传至中央服务器。
4.  **联邦聚合**：服务器采用加权平均等策略聚合各方权重，更新全局模型。
5.  **循环迭代**：重复上述过程，直至系统收敛，实现全局最优的缓存策略。

## 📊 仿真设置

项目支持通过 `args.py` 灵活配置仿真参数：

| 参数 | 说明 | 默认值 |
| :--- | :--- | :--- |
| `--n_client` | 边缘客户端（基站）数量 | 10 |
| `--n_content` | 系统总内容数量 | 3952 |
| `--Sn` | 单个基站的缓存容量 | 100 |
| `--T` | 仿真总时隙数 | 100 |
| `--lr` | 学习率 | 0.05 |
| `--epsilon` | $\epsilon$-greedy 策略参数 | 0.9 |

## 💻 快速开始

### 环境依赖

```bash
pip install torch pandas numpy
```

### 运行仿真

1.  **启动服务端 (CS)**：
    ```bash
    python CS--BS/main_FDRL.py
    ```
2.  **运行本地实验 (DQN2)**：
    ```bash
    cd DQN2
    python main_FDRL.py
    ```

## 📈 性能评估

系统主要评估指标为 **长期平均缓存命中率 (Long-term Average Cache Hit Ratio)**。通过对比联邦学习方案与传统独立 DQN 方案，验证协同学习在提升缓存效率方面的优越性。
